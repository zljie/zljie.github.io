# 统一认证平台 — 产品设计复盘

Published: February 2025

## 项目背景

在企业数字化建设过程中，随着业务系统的逐步增多，用户需要在多个系统间完成身份认证与授权管理。早期各系统独立建设，账户体系分散，导致了以下问题：

- **孤岛账户**：同一员工在多个系统中有独立账户，信息不同步，离职后需逐个清理
- **安全风险**：各系统认证机制参差不齐，部分系统缺少 MFA、强密码策略
- **合规挑战**：审计日志分散，无法形成完整的用户行为轨迹
- **管理成本**：新增系统时需重复接入，流程冗长

## 产品方案

### 核心模块设计

统一认证平台（IAM）作为企业身份基础设施，提供了五大核心模块：

#### 1. 用户管理中心

支持用户的全生命周期管理，包括：

- 员工入转调离流程自动化
- 组织架构同步（支持 HR 系统对接）
- 账户批量创建与导入
- 角色与权限批量分配

#### 2. 认证管理中心

提供多种认证能力的统一接入：

- **SSO 单点登录**：SAML 2.0、OAuth 2.0、OIDC 协议支持，统一各系统登录入口
- **多因素认证（MFA）**：短信、邮件、TOTP 动态口令等多种二次验证方式
- **自适应认证**：根据登录环境（IP、设备、地理位置）动态调整认证强度
- **社交登录**：支持企业微信、钉钉、飞书等国内主流 IM 集成

#### 3. 审计日志中心

完整的操作审计体系：

- 用户登录/登出的时间、设备、IP 记录
- 权限变更的完整变更链（who、when、what、from、to）
- 敏感操作的高亮标注（如管理员批量授权）
- 支持导出审计报告，满足合规要求

#### 4. 身份供应商管理（IdP）

支持对接外部身份源：

- 支持从上游 IdP（如企业微信）同步用户和组织信息
- 支持 SCIM 协议实现用户目录的自动化同步
- 支持多 IdP 场景下的身份桥接与映射

#### 5. 定制化模块

基于企业具体业务场景的扩展能力：

- 自定义登录页面与品牌主题
- 审批流程与认证策略的自定义编排
- 与企业工单系统、监控系统的集成

## 技术架构思路

### 以 OIDC 为核心的认证协议

选择 OIDC（OpenID Connect）作为主要认证协议，原因如下：

1. **基于 OAuth 2.0**：天然支持 API 授权，与现代云原生架构契合
2. **ID Token 携带用户信息**：减少用户信息查询的额外调用
3. **生态成熟**：主流语言和框架均有成熟的 OIDC 客户端库

### 微服务架构下的安全边界

各业务系统的认证模块下沉为基础设施，上层业务系统通过标准协议接入，无需关心认证细节。同时使用 API Gateway 统一处理认证令牌的验证，避免业务层的重复实现。

### 审计日志的异步写入

审计日志写入采用消息队列异步处理，避免同步写入带来的性能损耗。同时通过 Kafka 等消息中间件保证日志的可靠传输和持久化。

## 接入体验优化

### 快速接入套件

为降低接入方的工作量，提供了多语言的 SDK：

- 提供标准化的登录组件，前端可直接嵌入
- 后端验证 token 的工具库，支持主流语言
- 完整的接入文档与示例代码

### 沙箱环境

提供与生产环境隔离的测试环境，接入方可自由调试，配置变更不影响线上。

## 总结

统一认证平台的建设，本质上是将分散在各业务系统中的身份管理能力收敛，形成统一的基础设施。核心价值体现在：

- **降本**：减少重复建设和维护投入
- **安全**：统一安全策略，降低风险
- **合规**：完整的审计追溯能力
- **效率**：新系统接入周期从数周缩短至数天
