# BeEver 研发平台 — 低代码中台设计实践

Published: January 2025

## 项目背景

企业数字化进入深水区后，低代码平台已经成为加速业务交付的重要工具。然而市面上的低代码产品大多面向简单场景，对企业级复杂业务的支持不足：

- 表单驱动的低代码无法支撑多层级审批、条件流转等复杂逻辑
- SaaS 化的低代码平台在数据主权和定制化方面存在天然限制
- 业务人员与开发团队之间的协作仍存在鸿沟

BeEver 正是在这一背景下诞生的，目标是一个**能支撑企业级复杂业务**的低代码中台。

## 产品定位

BeEver 不是通用型的表单搭建工具，而是一个**元数据驱动的企业级低代码中台**。它的核心能力在于：

> 通过抽象、梳理、整合可复用的功能和场景，将其提炼为基础能力，并基于场景化组装式框架，帮助企业持续沉淀可复用的业务能力（PBC）。

## 核心架构：SCAF 场景化组装式应用框架

### 设计理念

传统低代码平台的核心单元是"组件"或"表单"，SCAF 的核心单元是**场景**。

- **组件**：原子级别的能力，如输入框、按钮、列表
- **场景**：业务级别的能力，如"房源登记"、"维修派单"、"合同审批"

每个场景封装了完整的业务逻辑（页面、流程、数据模型、权限），可以像乐高积木一样被组装。

### 三层抽象模型

```
┌─────────────────────────────────────────┐
│           场景层 (Scenario)              │  业务视角：维修派单、房源登记...
├─────────────────────────────────────────┤
│           能力层 (Capability)            │  可复用单元：审批流、数据校验...
├─────────────────────────────────────────┤
│           基础层 (Foundation)            │  技术视角：表单引擎、流程引擎...
└─────────────────────────────────────────┘
```

### 可视化编排

在 SCAF 框架下，业务人员可以通过可视化编辑器完成：

1. **场景选择**：从已沉淀的场景库中选择需要的业务场景
2. **能力组装**：通过拖拽配置场景间的依赖关系
3. **变量映射**：定义场景间的数据流转
4. **流程定制**：调整审批节点、流转条件

## 云原生架构

### 微服务拆分

平台采用微服务架构，每个核心能力独立部署：

| 服务 | 职责 |
|------|------|
| Scene Studio | 可视化编排服务 |
| Capability Hub | 场景和能力注册中心 |
| Runtime Engine | 场景运行时引擎 |
| Metadata Service | 元数据管理服务 |
| Form Engine | 表单渲染与数据处理 |
| Flow Engine | 流程引擎 |

### 容器云与 DevOps

- 所有服务容器化部署，支持 Kubernetes 编排
- 流水线自动化构建、测试、部署
- 蓝绿发布保障业务连续性

### 多租户隔离

作为中台产品，BeEver 需要服务多个租户。采用了**逻辑隔离 + 物理隔离**相结合的方案：

- 普通租户：共享基础设施，数据通过租户 ID 逻辑隔离
- 高安全要求的租户：独立数据库和计算资源

## 开发者体验

### 低代码与 pro code 结合

SCAF 解决了 80% 的通用场景，对于剩余 20% 的深度定制需求：

- 支持在场景中嵌入自定义组件（Vue/React）
- 支持通过代码扩展场景能力
- 场景导出为标准包，可在其他环境复用

### 版本管理与回滚

每个场景变更自动生成版本，支持：

- 版本的灰度发布（小流量验证）
- 一键回滚到历史版本
- 版本间的 diff 对比

## 总结

BeEver 的本质是**企业业务能力的平台化沉淀**。通过 SCAF 框架，将散落在各个系统中的业务逻辑抽象为可复用的场景，再通过可视化编排实现新业务的快速构建。这不仅提升了研发效率，更重要的是让企业的业务知识得以沉淀和传承。
