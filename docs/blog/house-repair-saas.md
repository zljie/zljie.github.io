# 房屋工单维修 SaaS 产品 — 低代码白标方案设计

Published: June 2024

## 项目背景

团队此前基于低代码平台构建了房屋维修管理系统，服务于企业内部业务团队。在实际运营中，我们发现：

1. **市场需求存在**：市场上缺乏针对房屋维修场景的垂直 SaaS 产品
2. **产品已验证**：内部使用期间，每套房屋维修成本下降了 30%
3. **差异化机会**：现有竞品多以通用工单系统为主，缺乏对维修场景的深度支持

基于此，我们决定将产品 SaaS 化，以白标（White Label）的方式输出给其他物业/地产企业。

## 核心挑战

从内部工具到 SaaS 产品，面临以下核心挑战：

### 多租户隔离

SaaS 产品的核心要求是多个租户共享基础设施但数据隔离。需要解决：

- 租户间的数据权限控制
- 配置信息的隔离存储
- 租户级别的资源配额管理

### 白标定制能力

不同企业有其独立的品牌诉求：

- 登录页面的 Logo、主题色定制
- 通知短信的发件人签名
- 特定字段的显示名称调整

### 版本发布策略

SaaS 产品需要持续迭代，同时保障存量租户的稳定性：

- 新功能的热插拔
- 租户可以选择升级时机
- 版本回滚的快速响应

## 技术方案

### 多租户架构设计

采用**共享数据库 + 租户 ID 隔离**的方案：

```
┌──────────────────────────────────────┐
│           API Gateway                │
│      (租户识别 & 路由分发)            │
├──────────────────────────────────────┤
│         Multi-tenant Service         │
│   ┌─────────┐ ┌─────────┐           │
│   │ TenantA │ │ TenantB │  ...       │
│   └─────────┘ └─────────┘           │
├──────────────────────────────────────┤
│        Shared Database               │
│   ┌─────────────────────────┐       │
│   │  tenant_id = 'A'         │       │
│   │  tenant_id = 'B'         │       │
│   └─────────────────────────┘       │
└──────────────────────────────────────┘
```

每个 API 请求通过中间件注入租户上下文，所有数据库操作自动带上租户过滤条件。

### 低代码白标引擎

白标配置通过**配置中心**统一管理：

```json
{
  "tenant": "enterprise-a",
  "branding": {
    "logo": "https://cdn.xxx.com/enterprise-a/logo.png",
    "primaryColor": "#1A73E8",
    "loginTitle": "企业A物业平台"
  },
  "features": {
    "enableSms": true,
    "enableWxNotify": false
  },
  "workflow": {
    "approvalSteps": 2,
    "autoAssign": true
  }
}
```

这些配置在运行时被注入到前端组件中，实现页面的动态定制。

### 打包与发布方案

设计了**单体部署**和**云服务模式**两套方案：

#### 单体部署方案

适用于对数据主权有强要求的企业：

- 提供 Docker 镜像，一键部署到客户私有环境
- 所有配置通过配置文件管理
- 客户自行维护升级

#### 云服务模式

适用于快速上线、无运维能力的企业：

- SaaS 托管模式，按租户数计费
- 自动热更新，客户无感知
- 提供 SLA 保障

### 打包工具链

开发了配套的打包 CLI：

```bash
# 构建白标包
beever build --tenant=enterprise-a --output=./dist/enterprise-a

# 验证包完整性
beever verify --package=./dist/enterprise-a/package.json

# 生成部署清单
beever manifest --package=./dist/enterprise-a
```

- 白标包支持 3 天内完成企业定制化配置
- 多租户架构支持单实例承载 100+ 租户
